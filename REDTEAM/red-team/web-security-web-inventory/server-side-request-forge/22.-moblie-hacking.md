## History of Android

* 2003: Android Inc. founded.
* 2005: Acquired by Google.
* 2007: Android 1.0 launched for developers.
* 2008: First Android smartphone (T-Mobile G1/HTC Dream) released.

### Version Highlights

* Cupcake (1.5): Introduced naming convention for versions.
* Donut (1.6): Support for CDMA networks.
* Éclair (2.0): Added text-to-speech, live wallpapers, and Google Maps Navigation.
* Froyo (2.2): Introduced Dalvik JIT compiler and Wi-Fi hotspot.
* Gingerbread (2.3): Refreshed UI and NFC support.
* Honeycomb (3.0): Optimized for tablets.
* Ice Cream Sandwich (4.0): Added facial recognition unlock.
* Jelly Bean (4.1): Improved notifications and browser support.
* KitKat (4.4): Optimized for low-RAM devices.
* Lollipop (5.0): Introduced Material Design.
* Marshmallow (6.0): Added fingerprint support and USB Type-C support.
* Nougat (7.0): Added Vulkan API.
* Oreo (8.0): Introduced Picture-in-Picture mode.
* Pie (9.0): Focused on battery optimization with machine learning.
* Android 10: Dropped dessert naming and introduced foldable phone support.
* Android 11: Added conversation notifications and built-in screen recording.

## Android Ecosystem

### Software Stack

* **Application Layer:**
  * Apps like Browser, Camera, and Calendar.
* **Application Framework:**
  * Components: Content Provider, Activity Manager, Notification Manager.
* **Libraries:**
  * Examples: libc, WebKit.
* **Android Runtime (ART):**
  * Features: Core libraries.
* **Hardware Abstraction Layer (HAL):**
  * Examples: Sensors, Camera, Bluetooth.
* **Linux Kernel:**
  * Drivers: Binder IPC, Shared Memory, Audio Driver, etc.

### Types of Applications

* **Native Apps:**
  * Developed using SDKs specific to Android (Java, Kotlin) or iOS (Swift, Objective-C).
  * Offer high performance but require platform-specific development.
* **Web Apps:**
  * Built using HTML5 and JavaScript.
  * Run in mobile browsers.
* **Hybrid Apps:**
  * Combine native and web technologies.
  * Run in WebView and can access device features.
* **Progressive Web Apps (PWAs):**
  * Built using modern web technologies.
  * Provide offline access and hardware integration.
