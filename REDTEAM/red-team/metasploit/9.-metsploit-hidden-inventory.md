## What & Why Metasploit?

* Founded by H.D. Moore in 2003 (developed in Perl), written in Ruby in 2007, acquired by Rapid7 in 2009, and released as Metasploit v6.0 in 2020
* The Metasploit Framework (MSF) is an open-source hacking and exploit development platform used to write, test, and execute exploit code
* Used to develop and test exploits
* Has a world database and public tested exploits
* Pre-installed in Kali Linux

## Metasploit Terminologies

| Term          | Description                                                                                               |
| ------------- | --------------------------------------------------------------------------------------------------------- |
| Interface     | Methods of interacting with the Metasploit Framework (`msfconsole`, Metasploit cmd)                       |
| Module        | Pieces of code that perform a particular task (an exploit)                                                |
| Vulnerability | Exploitable flaw or weakness in a computer system or network                                              |
| Exploit       | Code/module used to take advantage of a vulnerability                                                     |
| Payload       | Piece of code delivered to the target by an exploit (execute arbitrary commands or provide remote access) |
| Listener      | Utility that listens for an incoming connection from a target                                             |

## Metasploit’s Types

* **Auxiliary**
  * Scanning, fuzzing, sniffing, and admin capabilities
  * Offer extra assistance and functionality
  * Basically an active scanner / automated script like Nmap which helps in information gathering, scanning, and enumeration
* **Encoders**
  * Ensure that payloads are intact to their destination
  * Encoders are encoded using algorithms
* **Exploits**
  * Defined as modules that exploit a vulnerability and allow payload delivery
  * Basically take advantage of a vulnerability and execute the malware
* **NOPs**
  * No Operation code
  * Keep the payload sizes consistent across exploit attempts
  * Bypass IPS/IDS and antivirus
* **Payloads**
  * Code runs remotely and calls back to the attacker machine to establish a connection (or shell)
* **Plugins**
  * Additional scripts can be integrated within an assessment with `msfconsole` and coexist
* **Post**
  * Wide array of modules to gather information, pivot deeper, etc.
  * Exploit the POST will help in maintaining access
* **Evasions**
  * Invading the firewall

### What & Why Modules & Variables?

An MSF module requires additional information that can be configured through the use of MSF variables, both local or global variables, called options inside `msfconsole`/`msfdb`.

### What is Stage / Stagless

* Stage will cook the payloads in stage `_`
* Stagless will cook the payloads all together `/`
* Stagless payload it can be in EXE, C, PYTHON and that exe would have every functionality to communicate with the attacker
* Stage: it requires Metasploit to interact
* Stagless: everything is there in that payload
* Stage: not everything is there in that payload; the rest payloads will be available when connected to attacker
* Like the name speaks, it will be in stages like stage1, stage2, and more

Attacker generates stage version EXE to victim. Victim connects to attacker, and then attacker gives victim stage2, which has every functionality to communicate further.

* Stageless => exe => every functionality to communicate with the attacker
* Stage => only initial communication code will be included and the rest code will be supplied post connection
